#### 浏览器内核

`浏览器内核`是浏览器最重要的组成部分。

`浏览器内核`分两部分:`渲染引擎`和`JS引擎`

浏览器内核即`渲染引擎`主要用来解释网页语法并将网页渲染到屏幕上。由于不同的浏览器使用不同的内核,导致对网页语法解释不同，所以就会出现最终渲染的效果不同;常见浏览器内核:

| 浏览器类型 | 渲染引擎              | JS引擎         |
| ---------- | --------------------- | -------------- |
| IE         | Trident               | Chakra         |
| Chrome     | 以前Webkit, 现在Blink | V8             |
| Safari     | Webkit                | JavaScriptCore |

编程语言，主要分为编译型语言和解释型语言。

编译型语言的特点是在代码运行前编译器直接将对应的代码转换成机器码，运行时不需要再重新翻译，直接可以使用编译后的结果。（Java和 C++ ）
	解释型语言也是需要将代码转换成机器码，但是和编译型的区别在于运行时需要转换。比较显著的特点是，解释型语言的执行速度要慢于编译型语言，因为解释型语言每次执行都需要把源码转换一次才能执行。（ JavaScript和 ruby）

### 1. 浏览器的核心构成

浏览器是个复杂应用，主要由几大模块组成：

- 渲染引擎：负责解析 HTML、CSS，渲染出页面（比如 Chrome 的 Blink 引擎）。
- **JS 引擎**：专门处理 JavaScript 代码，执行逻辑、操作 DOM 等（Chrome 用的就是 **V8**）。
- 网络模块：处理 HTTP 请求、加载资源（如图片、脚本文件）。
- UI 界面：包括地址栏、标签页、右键菜单等可视化部分。

## 一、什么是 V8？

**V8** 是由 Google 开发的高性能 **JavaScript 与 WebAssembly 引擎**，使用 C++ 编写。它是 Chrome 浏览器和 Node.js 的核心组件之一。

- 实现 ECMAScript（即 JavaScript 标准）和 WebAssembly。
- 支持多种平台：Windows、macOS、Linux（x64/IA-32/ARM），以及社区维护的 IBM i、AIX、MIPS、ppcle64、s390x 等。
- 可嵌入任意 C++ 应用程序中。
- 提供完整的 JS 数据类型、操作符、对象、函数等标准实现（但不包含 DOM/BOM，这些由宿主环境如浏览器提供）。

 **关键特性**：

 - 即时编译（JIT）：通过 Ignition（解释器） + TurboFan（优化编译器）实现高性能执行。
 - 垃圾回收（GC）：精确、分代、Stop-the-world 的 GC 机制。
 - 内存管理：高效对象布局（Hidden Classes / Maps）、Slack Tracking 等优化。

## 二、V8 在浏览器中的角色

| 组件                | 职责                                      |
| ------------------- | ----------------------------------------- |
| **V8 引擎**         | 执行 JS 代码、管理对象内存、调用内置函数  |
| **Blink 渲染引擎**  | 解析 HTML/CSS、构建 DOM/CSSOM、布局与绘制 |
| **Chromium 浏览器** | 整合 V8 + Blink + 网络栈 + 安全模型等     |

> note：V8 **不提供 DOM API**。DOM 由 Blink（Chromium 的渲染引擎）提供，V8 仅负责执行 JS 并与宿主对象交互。

## 三、 v8执行流程

| 组件          | 功能简介                                                     |
| ------------- | ------------------------------------------------------------ |
| **Parser**    | JavaScript 源代码----词法/语法分析------->抽象语法树（AST）  |
| **Ignition**  | AST -->字节码（Bytecode）（解释器）                          |
| **TurboFan**  | 优化编译器（对热点代码生成高度优化的机器码）（热点代码是收集的性能数据的一种），若优化假设不成立，会触发反优化（Deoptimization） |
| **Heap & GC** | 管理对象的内存分配与垃圾回收                                 |
| **Snapshot**  | 引擎预初始化状态的快照，提高启动速度                         |

## 四、V8 对象模型

- **Hidden Classes（隐藏类 / Maps）**：V8 为每个对象动态生成“形状”（Map），加速属性访问。
- **对象内属性 vs 快属性 vs 慢属性**：根据属性数量和变化频率选择不同存储策略。
- **Slack Tracking**：在对象初始化阶段预留空间，避免频繁重分配。

## **GN 与 Ninja 是什么？（Chromium 构建体系的两大核心工具）**

构建 Chromium（Chrome 浏览器）涉及两个关键工具：

### 1️⃣ GN —— 生成构建配置文件的工具

**全称：Generate Ninja**
 路径：`src/tools/gn/` 或 `src/buildtools/linux64/gn`

你可以把 **GN 理解成“生成 Makefile 的工具”**。

### 🔍 GN 的作用：

- 读取 `BUILD.gn` 文件
- 解析依赖、源文件、编译选项
- 生成一个 **非常大的构建文件：`build.ninja`**
- 让 ninja 根据这个文件进行增量编译

**简单理解：**

> GN 决定 *“要编什么、怎么编”*。

你运行的命令：

```
gn gen out/Default
```

就是让 GN：

- 读取所有 `BUILD.gn`
- 根据 `args.gn` 配置生成 `out/Default/build.ninja`

------

### 2️⃣ Ninja —— 超快的编译器调度工具

官网描述：*small build system with a focus on speed*
 GN 生成的最终就是 **ninja 的输入文件**。

### 🔍 Ninja 的作用：

Ninja 类似传统 `make`，但是：

- 更快（几百万文件依赖也能秒级处理）
- 专为大型项目设计（Chromium / Android / V8 等）
- 支持极限并行

你执行的命令：

```
ninja -C out/Default chrome
```

就是让 ninja 按 **build.ninja** 的指令：

- 调用 Clang 编译 C++/Rust/JavaScript …
- 链接
- 构建 `chrome` 目标

**简单理解：**

> Ninja 决定 *“实际启动多少编译器、怎么调度编译任务、什么时候完成”*。

------

## 🧩 总结一句话

| 工具      | 作用                             | 你使用它时的命令              |
| --------- | -------------------------------- | ----------------------------- |
| **GN**    | 生成构建规则（生成 build.ninja） | `gn gen out/Default`          |
| **Ninja** | 根据 build.ninja 实际执行编译    | `ninja -C out/Default chrome` |
