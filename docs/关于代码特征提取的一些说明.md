#### V8 特征提取

现在，日志中会记录每个文件的 12 个代码特征：最大 AST 深度，最大迭代深度，函数数目（含最外层根函数），符号总数，节点总数，圈复杂度总和，最大圈复杂度，函数形参总数，分支总数，最大分支数，比较运算符总数，操作符总数。最新版本已经 push 到 gitlab 上 v8 的 **functionwise-feature-extraction** 分支。

可以在 feature-vector.h 中设置 filename 变量，以指定日志存放的路径。

调用 V8 时，采用 --no-lazy 参数以获取完整 AST 和代码特征。

#### 后处理脚本

postprecessor.py 是一个处理上述日志、生成网页粒度代码特征的脚本。使用 **python postprecessor.py PATH1 PATH2**，以对路径为 PATH1 的日志完成后处理，将结果存放到路径为 PATH2 的文件中。

postprecessor.py 会将一个日志内各个文件的代码特征汇总（“Max XX” 取各文件最大值，"Total XX" 取各文件之和），得到 json 格式的网页粒度代码特征，并清空日志，这样每次打开一个网页就会产生一次日志。对于一个测试网页，**首次访问**时，应当**启用 --no-lazy 参数**，在关闭网页后调用 postprecessor.py，指定合适的路径，获取它输出的 json（可以根据每个网页的 url 为该网页建立一个文件）；随后对网页性能的测试**不需要启用 --no-lazy 或调用 postprecessor.py**。

"Average XX" 或许是比 "Total XX" 更好的指标，将某个总和指标除以函数数目即可得到平均指标。
