## JavaScript 代码特征提取流程

1. JavaScript 文件在 V8 引擎上的运行流程为解析、解释执行、编译执行，其中后两个阶段可能随着优化 - 去优化的发生而不断切换。我们希望从解析代码得到的 AST 中实现特征提取，建立代码特征与最佳编译参数之间的联系。
2. 输入代码经过 Parser 解析得到 AST，V8 在 src/ast 下提供了 ast-traversal-visitor.h 头文件，可以继承并重写其中的 `AstTraversalVisitor` 类来遍历 AST 并收集一些信息。
3. 比较重要的特征可能包括：圈复杂度、最大循环嵌套深度、条件分支密度、平均函数大小、内联候选函数比例、热循环识别、闭包使用密度、动态属性访问比例、数组操作方法分布、类型转换操作密度、DOM 操作密度、事件监听器数量、异步操作模式、API 调用模式、第三方库使用检测等。前 10 个特征大多与 V8 的某些优化策略和编译参数密切相关，后 5 个则是与网页调优这一主题相契合的浏览器端特定特征。此外，JSTuner 文章中也给出了它选择的一些代码特征，可供参考。

## 当前进展

经过进一步的研究，以函数为粒度测算各类代码特征是最优的选择，我们考虑测量各函数内部的圈复杂度、条件分支密度等信息，并加以整理，形成程序级别的特征向量。

此外，JSTuner 文章中研究的是对神经网络程序（SIAMESE NEURAL NETWORK）的代码特征提取，其具有计算密集的特点，因此选取了众多硬件以及运行时相关的特征，以及许多和代码体积有关的粗粒度特征。这与我们研究的浏览器代码有一定不同之处。

以下是 `function-feature-ast-visitor.h` 的初步设计（以圈复杂度为例）：

```cpp
// New AstVisitor in order to extract function features

#ifndef V8_FUNCTION_FEATURE_AST_VISITOR_H_
#define V8_FUNCTION_FEATURE_AST_VISITOR_H_

#include <stack>

#include "src/ast/ast-traversal-visitor.h"

namespace v8 {
namespace internal {


class FunctionFeatureAstVisitor final
    : public AstTraversalVisitor<FunctionFeatureAstVisitor> {
 public:
  FunctionFeatureAstVisitor(uintptr_t stack_limit, Expression* root,
                        FunctionFeature* function_feature);

 private:
  friend class AstTraversalVisitor<FunctionFeatureAstVisitor>;

  void VisitSwitchStatement(SwitchStatement* stmt);
  void VisitIfStatement(IfStatement* stmt);
  void VisitForStatement(ForStatement* stmt);
  void VisitForInStatement(ForInStatement* stmt);
  void VisitForOfStatement(ForOfStatement* stmt);
  void VisitWhileStatement(WhileStatement* stmt);
  void DoWhileStatement(DoWhileStatement* stmt);
  void VisitConditionalChain(ConditionalChain* expr);
  void VisitConditional(Conditional* expr);
  void VisitTryCatchStatement(TryCatchStatement* stmt);
  void VisitTryFinallyStatement(TryFinallyStatement* stmt);

  void VisitFunctionLiteral(FunctionLiteral* expr);

  FunctionFeature* function_feature_ = nullptr;
  std::stack<int> stk;
};

}  // namespace internal
}  // namespace v8

#endif  // V8_FUNCTION_FEATURE_AST_VISITOR_H_
```

在遇到对应的 `stmt` 时，当前所处函数的圈复杂度增加对应的值；遇到 `FunctionLiteral* expr` 代表进入新的函数，由于 JavaScript 中函数可以嵌套定义，采用一个栈来记录目前位置所在的各层函数；`FunctionFeature` 类记录该程序中所有函数的特征，其中的每个元素形如 (id, func_name, cycl, braches, func_size, ...)，向 `FunctionFeatureAstVisitor` 中传入一个指针以存放返回值。

## 研究编译选项之间的相关性

与具体要执行的代码相独立地，有些编译选项之间呈现正相关 / 负相关等特定关系时，执行性能较好。对这一关系的探究可以在测算 Example - Best Compiling Flags 的同时实现（因为该过程需要在多个不同编译参数的组合下测量数据），通过一些模型从这些数据中提取较为显著的结果。
