
## 第 2 步：安装 depot_tools（Google 的构建工具集）

> depot_tools 是用来获取和编译 V8、Chromium 等项目的官方工具。

**拉取v8源码的两种方式：**

①从googlesource官网安装，一般会遇到网络问题

②手动git clone官方镜像Github仓库

下面采用方法②：

```
cd ~
git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git
echo 'export PATH="$PATH:$HOME/depot_tools"' >> ~/.zshrc
source ~/.zshrc
```

验证是否安装成功：

```
which fetch
```

若能输出路径（如 `/home/你的用户名/depot_tools/fetch`），表示成功。

## 绕过代理克隆仓库(解决gclient无法执行问题)

问题：fetch v8从google官网访问，网络问题始终无法解决

`git clone https://github.com/v8/v8.git`

# 网络问题无法用 fetch，也可以手动等价操作

有时候国内网络访问 Google 源不通，这时可以：

```
# 1. 手动 clone
git clone https://github.com/v8/v8.git
cd v8

# 2. 手动配置 .gclient
# 在~主目录下执行（v8的上一级目录）
gclient config --name v8 https://chromium.googlesource.com/v8/v8.git

# 3. 同步依赖
gclient sync
```

gclient sync报错：源地址不一致

URL: https://chromium.googlesource.com/v8/v8.git (git)

The local checkout in /home/rmy/v8 reports:
https://github.com/v8/v8.git (git)

```go
# 解决办法：更换远程仓库
➜  v8 git:(main) ✗ git remote set-url origin https://chromium.googlesource.com/v8/v8.git
➜  v8 git:(main) ✗ git remote -v
origin  https://chromium.googlesource.com/v8/v8.git (fetch)
origin  https://chromium.googlesource.com/v8/v8.git (push)
```

注：单纯的git clone只是拉取v8源码，但是还有各种依赖需要gclient sync同步安装。

| 对比项               | `fetch v8`                                           | `git clone v8.git`                                 |
| -------------------- | ---------------------------------------------------- | -------------------------------------------------- |
| **是否只下 v8 目录** | ❌ 会创建完整目录结构（v8 + build + third_party 等）  | ✅ 只拉 v8 目录                                     |
| **是否包含依赖**     | ✅ 自动下载所有依赖（build system、gn、ninja 脚本等） | ❌ 没有，需要手动 `gclient config` + `gclient sync` |
| **配置文件生成**     | ✅ 自动创建 `.gclient` 文件（用于依赖管理）           | ❌ 无，必须手动创建                                 |
| **更新方式**         | `gclient sync`（同步所有依赖）                       | 手动 `git pull`（只能更新主仓库）                  |
| **能否直接编译**     | ✅ 一下子就能 `v8gen.py + ninja` 编译                 | ❌ 编译会报错：缺少依赖目录                         |
| **适用场景**         | ✅ 初次下载或环境配置                                 | ⚙️ 已经有完整环境，只想快速拉主仓库的特定版本       |

【注】**以下均在`/v8/out/x64.release`目录下执行（d8根目录）**

- 可以测试运行 JS 脚本

进入输出目录并运行：

```
cd ~/v8/out/x64.release
./d8 -e "print('Hello, V8!')"
```

输出应为：

```
Hello, V8!
```

- 运行一个 JS 文件

```
echo "print(1 + 2 * 3);" > test.js
./d8 test.js
```

输出：

```
7
```

- 查看构建参数

如果你想看 `gm.py` 自动生成的构建配置：

```
cat ~/v8/out/x64.release/args.gn
```

- **手动修改并重新编译**

启用调试信息或符号表：

```
gn args out/x64.debug
```

添加：（vim编辑器 esc退出，输入:wq，w写入，q退出）

```
is_debug = true
symbol_level = 2
```

再编译：

```
autoninja -C out/x64.debug d8
```
